# Paddle Setup Guide for AudioFile

## Overview
AudioFile uses Paddle as the payment processor for Premium subscriptions. Paddle handles:
- Payment processing
- Tax compliance (global VAT/GST)
- Subscription management
- Customer portal
- Webhook events for subscription lifecycle

## Step 1: Create Paddle Account

1. Sign up at [paddle.com](https://paddle.com)
2. Complete business verification (takes 1-3 business days)
3. Choose your account type:
   - **Sandbox**: For testing (recommended first)
   - **Production**: For live payments

## Step 2: Create Products in Paddle Dashboard

### Premium Monthly Subscription
1. Go to **Products** in Paddle dashboard
2. Click **Add product**
3. Fill in:
   - **Name**: Premium Monthly
   - **Description**: Unlimited collections, wishlists, and sharing
   - **Tax category**: Digital goods / Software as a Service
4. Add a price:
   - **Type**: Recurring
   - **Amount**: $5.00 USD
   - **Billing period**: Monthly
5. Save and copy the **Price ID** (format: `pri_01hxyz...`)

### Premium Annual Subscription (Optional)
1. Create another product: **Premium Annual**
2. Price: $49.00 USD / Year
3. Copy the Price ID

## Step 3: Configure Webhooks

1. Go to **Developer Tools** > **Notifications**
2. Click **Add notification destination**
3. Set:
   - **URL**: `https://audiofile.app/api/billing/webhook`
   - **Events to subscribe to**:
     - `transaction.completed`
     - `subscription.created`
     - `subscription.updated`
     - `subscription.canceled`
     - `subscription.paused`
4. Copy the **Webhook Secret** (format: `pdl_ntfset_...`)

## Step 4: Get API Credentials

1. Go to **Developer Tools** > **Authentication**
2. Generate an API key
3. Copy the **API Key** (format: `pdl_live_...` or `pdl_test_...`)

## Step 5: Configure Environment Variables

Add these to your production environment (Coolify):

```bash
# Paddle API Configuration
PADDLE_API_KEY=pdl_live_XXXXXXXXXXXXXXXXXXXXX
PADDLE_WEBHOOK_SECRET=pdl_ntfset_XXXXXXXXXXXXXXXXXXXXX
PADDLE_ENVIRONMENT=production

# Application URLs
APP_BASE_URL=https://audiofile.app
```

For sandbox testing:
```bash
PADDLE_API_KEY=pdl_test_XXXXXXXXXXXXXXXXXXXXX
PADDLE_WEBHOOK_SECRET=pdl_ntfset_XXXXXXXXXXXXXXXXXXXXX
PADDLE_ENVIRONMENT=sandbox
```

## Step 6: Update Frontend with Price IDs

Edit `frontend/src/components/BillingSettings.svelte`:

```typescript
// Replace this placeholder with your actual Paddle price ID
const PREMIUM_MONTHLY_PRICE_ID = 'pri_01hxyz...'; // Your real price ID here
```

If you want to offer annual plans, you'll need to add UI for that as well.

## Step 7: Configure Customer Portal

1. Go to **Customer Portal** in Paddle dashboard
2. Enable the portal
3. Configure:
   - **Branding**: Add your logo and colors
   - **Features**: Enable subscription management, payment method updates
   - **Return URL**: `https://audiofile.app/settings/billing`

The portal URL is automatically generated by the backend using the customer ID.

## Step 8: Test in Sandbox Mode

1. Deploy the app with sandbox credentials
2. Create a test account
3. Click "Upgrade to Premium"
4. Use Paddle's test card numbers:
   - **Success**: `4242 4242 4242 4242`
   - **Decline**: `4000 0000 0000 0002`
5. Verify:
   - Checkout page loads correctly
   - Payment processes successfully
   - Webhook fires and updates subscription status
   - User gets Premium access
   - Can access customer portal

## Step 9: Go Live

1. Switch environment variables to production credentials
2. Redeploy the app
3. Test with a real payment (small amount, then refund)
4. Monitor webhooks in Paddle dashboard
5. Verify subscription appears in database

## Step 10: VIP Exemptions

To exempt specific users from payment requirements (wife, friends, early testers):

```sql
-- Run in Supabase SQL editor
UPDATE profiles 
SET is_vip = true 
WHERE id = 'user-uuid-here';
```

Or use the admin API (requires admin account):
```bash
curl -X POST https://audiofile.app/api/admin/billing/vip \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"userId": "user-uuid-here", "isVip": true}'
```

## Webhook Event Flow

### Successful Subscription
1. User clicks "Upgrade to Premium"
2. Frontend calls `POST /api/billing/checkout` with price ID
3. Backend creates Paddle transaction and returns checkout URL
4. User completes payment on Paddle's hosted checkout
5. Paddle sends `transaction.completed` webhook
6. Backend updates subscription status to `active`
7. User now has Premium access

### Subscription Cancellation
1. User cancels in Paddle customer portal
2. Paddle sends `subscription.canceled` webhook
3. Backend updates status to `canceled`
4. User retains access until `current_period_end`
5. After period ends, access reverts to Free tier
6. Existing collections/wishlists are preserved (soft downgrade)

### Failed Payment
1. Paddle retries payment automatically
2. If retries fail, Paddle sends `subscription.paused` webhook
3. Backend updates status to `paused`
4. User sees warning banner to update payment method
5. If payment succeeds, subscription reactivates
6. If payment fails after grace period, subscription cancels

## Troubleshooting

### Webhook Not Firing
- Check webhook URL is correct: `https://audiofile.app/api/billing/webhook`
- Verify webhook secret matches in environment variables
- Check server logs for webhook errors
- Use Paddle's webhook testing tool in dashboard

### Checkout Not Working
- Verify Paddle API key is correct
- Check price ID exists in Paddle dashboard
- Ensure price ID is active (not archived)
- Check browser console for API errors

### Subscription Not Updating
- Verify webhook events are enabled in Paddle dashboard
- Check database for subscription record: `SELECT * FROM subscriptions WHERE user_id = '...'`
- Check server logs for webhook processing errors
- Verify `custom_data.user_id` is being passed correctly

## Monitoring

### Application Logs
Monitor for:

- Paddle API errors
- Database update failures
- Missing user_id in custom_data

### Database Queries
```sql
-- Active premium subscriptions
SELECT COUNT(*) FROM subscriptions WHERE status = 'active';

-- Users over free tier limits
SELECT user_id, COUNT(*) as item_count 
FROM collection_items 
GROUP BY user_id 
HAVING COUNT(*) > 50;

-- VIP users
SELECT id, email FROM profiles WHERE is_vip = true;
```

## Cost Analysis

### Paddle Fees
- **5% + 50¢ per transaction**
- Example: $5 subscription = $0.75 fee = $4.25 to you (85%)

### Why Paddle Over Stripe
- **Tax compliance**: Paddle handles global VAT/GST automatically
- **Dispute handling**: Paddle fights chargebacks for you
- **Simpler integration**: 10-20 hours vs 40-80 hours for Stripe
- **Time savings**: 20+ hours/month on tax compliance
- **At 300 users**: Paddle saves ~$1,961/month when valuing your time at $100/hr

See `docs/strategy/payment-processor-deep-dive.md` for full analysis.
